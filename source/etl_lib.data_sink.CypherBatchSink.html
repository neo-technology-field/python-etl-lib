  • task (Task) – etl_lib.core.Task.Task instance owning this batchProcessor.

  • predecessor (BatchProcessor) – BatchProcessor which get_batch() function will be called to receive batches to process.

  • query (str) – Cypher to write the query to Neo4j. Data will be passed as batch parameter. Therefore, the query should start with a UNWIND $batch AS row.
